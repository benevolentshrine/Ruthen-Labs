BORU CLI COMMANDS REFERENCE
===========================

BORU — Security Cage Engine for Project MOMO
"What runs here, stays here."

COMMANDS & PURPOSES:

1. cage
   Description: Execute code in the security cage.
   Purpose: This is BORU's core feature. It intercepts and sandboxes AI-generated code (via WASM) before it can touch your system. It acts as a security membrane, blocking unauthorized syscalls, file access, and network calls based on the selected security mode.
   Arguments:
     --input, -i <PATH>    Input file to execute
     --mode <MODE>         Security mode: hard, mid (default), easy, audit, custom
     --fuel <N>            Fuel limit (max WASM instructions)
     --config <PATH>       Load custom rules from config file

2. check
   Description: Static analysis check (dry-run).
   Purpose: Performs static analysis on a binary (like WASM) to inspect for dangerous patterns or unauthorized imports without actually executing the code. Useful for a quick safety check.
   Arguments:
     --input, -i <PATH>    Input file to analyze

3. daemon
   Description: Start the socket daemon.
   Purpose: Runs BORU as a background service that listens on a Unix socket. This allows other parts of Project MOMO (like AI agents) to send execution requests to BORU programmatically.
   Arguments:
     --socket <PATH>       Socket path (optional)

4. tui
   Description: Launch the Ratatui dashboard.
   Purpose: Provides a real-time visual dashboard for monitoring the BORU engine. It displays live audit events, system status, and interception activity in a terminal-based UI.
   Arguments:
     --socket <PATH>       Socket path to connect to

5. log
   Description: View audit logs.
   Purpose: Manages the tamper-proof SHA-256 audit trail. Every intercepted action or blocked operation is recorded here. This command allows you to view, tail, filter, and verify the integrity of these logs to ensure they haven't been tampered with.
   Arguments:
     --tail                Tail the log continuously
     --severity <LEVEL>    Filter by severity level
     --since <TIME>        Show entries since timestamp
     --export <PATH>       Export logs to file
     --clear               Clear all audit logs (with confirmation)
     --verify              Verify the tamper chain integrity
     --entry <N>           Verify a specific entry by sequence number

6. quarantine
   Description: List quarantined files.
   Purpose: Manages files that BORU has flagged as malicious and moved to an isolated directory. It allows you to inspect what was blocked, restore files if they were false positives, or permanently delete threats.
   Arguments:
     --list                List all quarantined files
     --restore <ID>        Restore a quarantined file
     --delete <ID>         Delete a quarantined file permanently

7. deps
   Description: Show dependency status for all runners.
   Purpose: Checks which runtime interpreters and tools (like wasmtime, python3, node) are available on the system. This helps ensure that BORU has the necessary environment to execute and sandbox different types of code.

8. scan
   Description: Scan a directory for threats.
   Purpose: Performs a batch scan of all files in a directory to detect potential threats using entropy analysis and hash database matching. Useful for checking a project's safety before starting work.
   Arguments:
     --path, -p <PATH>     Path to scan
     --mode <MODE>         Security mode: hard, mid (default), easy, audit
     --depth <N>           Maximum recursion depth
     --format <FORMAT>     Output format: console (default), markdown
     --report <PATH>       Export report to file

9. watch
   Description: Start watchdog mode (real-time file monitoring).
   Purpose: Actively monitors a directory for file changes (new files, modifications). It automatically scans any changed files in real-time and can quarantine them immediately if they violate security policies.
   Arguments:
     --path, -p <PATH>     Path to watch
     --mode <MODE>         Security mode: hard, mid (default), easy, audit
     --recursive <bool>    Watch recursively (default: true)
     --poll                Use polling fallback instead of native events

10. db
    Description: Manage the hash database.
    Purpose: Manages a local database of SHA-256 hashes for known-bad files. BORU uses this database during scans and watchdog monitoring to instantly identify and block recognized malware or dangerous scripts.
    Arguments:
      --add <SHA256> <NAME> Add a hash and name to the database
      --family <FAMILY>      Family for the hash entry
      --check <PATH>         Check a file's hash against the database
      --remove <SHA256>      Remove a hash from the database
      --import <PATH>        Import hashes from a JSON file
      --stats                Show database statistics
      --list                 List all entries

11. replay
    Description: Session replay — view execution timelines.
    Purpose: Allows for forensic analysis by replaying the exact sequence of events that occurred during a sandboxed execution session. This is vital for understanding exactly how a malicious script attempted to attack the system.
    Arguments:
      --list                 List all sessions
      --session <ID>         Session ID to replay
      --export <PATH>        Export session to file

12. rollback
    Description: Filesystem rollback.
    Purpose: Provides a "system restore" feature for AI execution. BORU creates shadow snapshots before execution, allowing you to undo any filesystem changes (created files, deletions, modifications) made by an AI agent if they turn out to be destructive.
    Arguments:
      --list                 List all shadow sessions
      --session <ID>         Session ID to rollback
      --dry-run              Dry-run (show what would be restored)
      --clear <ID>           Clear shadow for a session

13. iam
    Description: Agent identity management.
    Purpose: Manages access control and tokens for AI agents. Only authorized agents with a valid token can communicate with the BORU daemon. This prevents unauthorized processes from using the security cage to hide malicious activity.
    Arguments:
      --create-agent <NAME>  Create a new agent
      --description <DESC>   Agent description (default: AI coding agent)
      --list                 List all agents
      --revoke <NAME>        Revoke an agent's token
      --show <NAME>          Show agent details
